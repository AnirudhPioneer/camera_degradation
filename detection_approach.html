1Introduction & Goals

This system detects when the optical path of an externally mounted camera has become unreliable — due to weather, physical contamination, or partial blockage. It is a detection-only system. It does not attempt to restore, dehaze, or recover any image information. Its sole job is to raise an alarm so that the host system can respond appropriately (e.g. stop relying on the camera feed, request a lens clean, or flag the data as invalid).

The target platform is an ARM Cortex-M0 microcontroller embedded in a camera ISP — a processor with no floating-point unit, no division in hardware, very limited RAM (~2 KB budget for this module), and no operating system. This makes the algorithm design radically different from what you would write for a laptop or GPU. Every design decision has been made with these constraints at the forefront.

Design contract

No float or double anywhere. No dynamic memory allocation. No standard library beyond stdint.h and string.h. No printf in the core module. All buffer sizes are compile-time constants. Integer overflow is explicitly managed at every risk point.

The outputs are one of six classes:

CLEAN  FOG  FROST  DIRT  OBSTRUCTION  UNKNOWN

2What We Detect and Why

The Degradation Types

Each degradation leaves a distinct fingerprint on the image. Understanding the physical cause of each helps explain why the features were chosen.

FOG & Atmospheric Haze

Fog scatters light uniformly across the entire field of view. Every part of the image loses contrast and sharpness by roughly the same amount — distant objects disappear first, but the degradation is spatially even. The histogram is compressed toward mid-tones. The global mean stays close to the scene's normal level because the fog adds scattered light but also attenuates the original signal, roughly balancing out.

FROST & Condensation

Ice crystals or condensation on the lens surface create a diffuse, semi-translucent coating directly on the optical element. Unlike fog, the scattering happens right at the lens — so it degrades the image uniformly but adds a bright veil on top of the scene, pushing the global mean brightness noticeably higher. The histogram collapses toward the high end (everything washes out bright). Sharpness collapses completely because the ice crystals are directly in the focal path.

DIRT & Mud

An opaque dark blob sitting on part of the lens. Unlike fog or frost, dirt is spatially localised — it darkens specific grid cells while leaving others unaffected. The affected cells show near-zero internal variance (a solid mud blob has no texture) and their average brightness is significantly lower than the rest of the frame. The unaffected cells look normal, so the overall image may still have reasonable global contrast and sharpness.

OBSTRUCTION

Any solid object placed on or very close to the lens that blocks part of the field of view. Unlike dirt, the obstruction may not be dark — tape, a sticker, an ice chunk, or a bright reflective surface all create a flat (low-variance) region that can be at any brightness. The detection logic is the same as for dirt (look for flat cells with spatial non-uniformity) but without the dark-brightness requirement.

3The Detection Pipeline

Single-Pass Design

On a PC you would typically write separate functions for each metric, looping over the image multiple times. On a Cortex-M0 with a cache-less memory bus, every pass over a large image is expensive. We make one pass over the image and compute all metrics simultaneously. The pixel at position (x, y) contributes to the Laplacian, the histogram, the global statistics, and its grid cell in the same inner loop iteration.

Input: Grayscale Frame (uint8, row-major)
H × W bytes — no padding, values 0–255
Single Pass at Stride 4
Visits 1/16th of all pixels. Computes simultaneously: Laplacian, min/max/sum, histogram bins, grid cell accumulators.
Thumbnail MAD (192 pixel reads, separate micro-loop)
Reads 16×12 thumbnail, computes inter-frame difference vs. stored previous thumbnail, updates stored thumbnail.
Feature Derivation
From accumulators: laplacian_var, global_contrast, global_mean, histogram_spread, occlusion_score, obstruction_score, cell_mean_variance, temporal_mad
EMA Baseline Update
Fast EMA (shift 4, ~16 frame time constant) always updates. Slow EMA (shift 8, ~256 frames) only updates when classification is CLEAN.
Ratio Computation
For each feature: ratio = (fast_raw × 128) / (slow_raw + 1). Value of 128 = at baseline. Below = dropped. Above = elevated.
Decision Tree
Ratios evaluated in priority order → raw classification label + confidence 0–255
Hysteresis State Machine
Enter degraded after 5 consecutive detections. Clear to CLEAN after 15 consecutive CLEAN frames. Output: final DegradationClass

Step 1 — Spatial Subsampling (Stride 4)

The main loop visits every 4th pixel in both x and y directions, starting 1 pixel in from the border (so the Laplacian filter can safely read neighbouring pixels without an out-of-bounds check).

// C — degradation.c
for (y = 1; y < height - 1; y += DEGR_STRIDE) {
    for (x = 1; x < width - 1; x += DEGR_STRIDE) {
        uint8_t p = frame[y * width + x];
        /* ... all metrics updated here ... */
    }
}

For a 640×480 frame this visits about 19 000 pixels instead of 307 200 — a 16× reduction. For the kind of degradations we care about (fog, frost, dirty lens) the spatial detail is irrelevant. What matters is the statistical distribution of pixel values, and a 1-in-16 sample is more than sufficient for that.

Why not downsample first?

Downsampling with interpolation requires an intermediate buffer (the resized image). We don't have the RAM for it. Stride-4 access achieves the same coverage in a single pass with zero extra storage. The trade-off is that the Laplacian reads full-resolution neighbours — it measures fine-detail sharpness correctly — whereas a downsampled image would measure only coarser-scale edges.

Step 2 — Laplacian Variance (Sharpness Proxy)

The Laplacian is a second-order derivative filter. Applied to a pixel, it measures how different that pixel is from its immediate neighbours. On a sharp edge the Laplacian is large; on a smooth or blurred region it is near zero. We use the simplest 3×3 discrete approximation:

kernel:     [ 0,  1,  0 ]
            [ 1, -4,  1 ]
            [ 0,  1,  0 ]

lap = 4×p − p_above − p_below − p_left − p_right

Because we only care about the magnitude of sharpness (not the direction of edges), we take the absolute value |lap| and clamp it to 255. We then accumulate the sum of squares and divide by the pixel count, giving us the mean squared Laplacian response — a number that is high for a sharp image and near zero for a blurred one.

// C — degradation.c
int16_t lap = (4 * (int16_t)p)
            - frame[(y-1) * width + x]
            - frame[(y+1) * width + x]
            - frame[y * width + (x-1)]
            - frame[y * width + (x+1)];
if (lap < 0)   lap = -lap;
if (lap > 255) lap = 255;  /* clamp before squaring — prevents uint32 overflow */
lap_sumsq += (uint32_t)lap * lap;
lap_count++;

After the loop: laplacian_var = (lap_sumsq / lap_count) >> 4. The right-shift by 4 scales the result into the 0–255 range — a typical outdoor sharp scene reads around 40–120; a fog-degraded scene drops below 15.

Overflow management

Clamping |lap| to 255 before squaring is critical. Without it, lap² can reach 1020² = 1 040 400, and summing ~4500 such values would overflow a uint32_t. After clamping, the maximum sum is 4500 × 255² = 292 million, safely below 2³² ≈ 4.3 billion.

Step 3 — Michelson Contrast

Michelson contrast is a global measure of the dynamic range of pixel values in the frame. It compares the brightest sampled pixel to the darkest:

global_contrast = (max − min) × 256 / (max + min + 1)

The +1 in the denominator prevents division by zero when both max and min are zero (completely black frame). The ×256 stretches the result across the full uint8 range. On a normal scene this gives values around 100–200. Fog compresses the dynamic range — the darkest and brightest pixels converge toward mid-grey — so this number drops. Frost has a similar effect. Dirt or obstruction covering only part of the lens may not affect global contrast much.

Critically, we accumulate global_min and global_max inside the main loop with simple comparisons — zero extra storage, computed for free alongside everything else.

Step 4 — Histogram Spread & Global Mean

We build an 8-bin histogram of pixel intensities. Each bin covers 32 intensity values (0–31 → bin 0, 32–63 → bin 1, …, 224–255 → bin 7). The bin index for a pixel is simply pixel_value >> 5 — a right-shift, no division, no lookup table.

After the loop, we compute the histogram spread: the distance between the 5th-percentile bin and the 95th-percentile bin, multiplied by 32. This tells us how wide the pixel value distribution is.

// Walk from the left until we've seen 5% of pixels
target = global_count / 20;
cumul = 0;
for (b = 0; b < 8; b++) {
    cumul += hist[b];
    if (cumul > target) { bin_lo = b; break; }
}
// Walk from the right until we've seen 5% of pixels from the top
cumul = 0;
for (b = 7; b >= 0; b--) {
    cumul += hist[b];
    if (cumul > target) { bin_hi = b; break; }
}
histogram_spread = (bin_hi > bin_lo) ? (bin_hi - bin_lo) * 32 : 0;

The spread is the key discriminator between fog and frost:

  • A normal outdoor scene: spread ≈ 160–224 (pixels span most of the range)
  • Fog: spread drops to 64–96 (mid-tones dominate, extremes disappear)
  • Frost: spread drops below 64 (everything washes out to bright, extremely compressed)

The global mean is accumulated as a running sum divided by the sample count at the end. It is the other frost discriminator — fog keeps mean near baseline, frost elevates it significantly because the ice coating adds a bright diffuse layer on top of the scene.

Step 5 — Grid Cell Analysis (Occlusion & Obstruction)

The frame is divided into a 4-column × 3-row grid of 12 cells. For each cell we accumulate the sum of pixel values and the sum of squared pixel values. After the loop, these give us the mean and variance for each cell.

Why a grid?

Global metrics (contrast, mean, histogram) cannot distinguish between a uniformly degraded frame and a partially blocked one. A piece of dark tape covering the bottom-right corner does not significantly change the global mean or contrast — but it does make one or two cells very dark and flat while the others remain normal. The grid provides spatial awareness at minimal memory cost.

Cell internal variance (flatness detection)

The variance of pixels within a single cell is computed as:

local_variance = (sumsq / count) − mean²
lvar8 = local_variance >> 8   /* scale to uint8 */

A cell is considered flat if lvar8 < 8. A flat cell means all pixels in that region are nearly the same value — the signature of something solid (mud, tape, ice) sitting on the lens. A normal scene has varied textures, so cell variance stays well above this threshold.

Occlusion score vs. Obstruction score

Occlusion (→ DIRT)

  • Flat cell (lvar8 < 8)
  • AND dark: cell mean < global mean / 2
  • Counts dark flat cells (0–12)
  • Signature of mud, dirt blobs

Obstruction

  • Flat cell (lvar8 < 8)
  • At any brightness
  • Counts all flat cells (0–12)
  • Includes bright obstructions (tape, ice)

Every occluded cell is also an obstructed cell, but not vice versa. The brightness requirement is what separates them.

Cell mean variance (spatial non-uniformity)

After computing all 12 cell means, we compute the variance of those means:

cell_mean_variance = variance(cell_mean[0..11]) >> 6

If all 12 cells have similar mean brightness, this value is low — indicating the frame is spatially uniform (typical of fog or frost, which affect the whole frame equally). If some cells are bright and others dark — as happens when part of the lens is covered — this value is high, indicating spatial non-uniformity (typical of dirt or obstruction).

This is the critical gate that separates lens obstruction from global degradation.

Step 6 — Temporal Thumbnail MAD

To detect whether the camera is seeing a static scene (which could indicate the lens is frozen or covered), we compare each frame to the previous one. Rather than comparing full frames (too much RAM), we work on a tiny 16×12 pixel thumbnail — 192 bytes.

The thumbnail is downsampled by mapping each of the 192 positions to the nearest pixel in the full frame. We compute the Mean Absolute Difference (MAD) between the current thumbnail and the one stored from the previous frame:

raw_mad = sum(|current[i] - previous[i]|) / 192

This value is smoothed through a fast EMA. A high MAD means the scene is changing (moving vehicles, people walking). A low, stable MAD means nothing is moving — which is consistent with degradation but also with a genuinely static scene. For this reason, temporal MAD is used as a supporting signal rather than a primary trigger. The dual-EMA baseline handles the static-scene ambiguity by comparing the current MAD rate to the historical baseline MAD rate.

4Dual-Timescale Baseline System

Exponential Moving Averages

The biggest challenge in degradation detection is that different scenes, lighting conditions, and camera placements produce wildly different "normal" values. A camera looking at a bright sunny road has very different sharpness, contrast, and mean brightness from one mounted in a shaded tunnel. If we used fixed absolute thresholds (e.g. "fog if contrast < 40") the detector would fire incorrectly in many legitimate environments.

The solution is to track a personalised baseline for each feature based on what the camera has seen when the scene was known to be clean. We maintain two baselines for each feature using Exponential Moving Averages (EMAs) at different speeds:

Fast EMA
α = 1/16
Right-shift by 4
Time constant ≈ 16 frames
Tracks recent conditions
Slow EMA
α = 1/256
Right-shift by 8
Time constant ≈ 256 frames
Tracks long-term baseline

The EMA update formula — implemented with integer bit-shifts rather than multiplication or division — is:

ema = ema − (ema >> shift) + raw_value

At steady state, the EMA converges to raw_value × (1 << shift). To recover the approximate raw value from the EMA: raw ≈ ema >> shift. This is why the EMAs are stored as uint16_t rather than uint8_t — they hold the raw value scaled up by 16 or 256, which requires 16-bit storage to avoid overflow.

Why bit-shifts instead of floating-point?

The Cortex-M0 has no hardware division and no FPU. A software floating-point divide on this processor takes hundreds of cycles. A bit-shift takes one cycle. By choosing alpha values that are exact negative powers of two (1/16, 1/256), we eliminate all multiplication and division from the baseline update — one of the most frequently executed code paths.

The Ratio — What 128 Means

Instead of comparing a feature's raw value against an absolute threshold, we compare the fast EMA against the slow EMA. The ratio is:

ratio = (fast_raw × 128) / (slow_raw + 1)

where fast_raw = fast_ema >> 4 and slow_raw = slow_ema >> 8.

Ratio valueMeaning
128Fast EMA equals slow EMA — feature is at its normal baseline
< 128Fast EMA has dropped below the slow EMA — feature has decreased recently
> 128Fast EMA has risen above the slow EMA — feature has increased recently
64Feature is at 50% of its baseline — a significant drop
192Feature is at 150% of its baseline — a significant rise

This design means the detector automatically adapts to any environment. Whether the baseline sharpness is 40 or 140, the ratio-based threshold fires when sharpness drops to a given fraction of whatever the clean-scene baseline happened to be. No per-deployment tuning required.

Slow EMA Freeze During Degradation

There is a subtle but critical problem with always-updating baselines: if a degradation persists for many frames, the slow EMA will eventually drift toward the degraded values, the ratio will return to 128, and the alarm will extinguish itself even though the lens is still dirty. This is exactly the failure mode observed in early testing.

The solution: the slow EMA is frozen as soon as a degradation is confirmed in the hysteresis state machine, and stays frozen until the classification returns to CLEAN.

if (state->hyst_class == DEGRADATION_CLEAN) {
    /* Both EMAs update freely — building the clean baseline */
    state->baseline.lap_slow = ema_update(state->baseline.lap_slow, ...);
} /* else: slow EMA stays locked at the last clean value */

/* Fast EMA always updates — it tracks the current (degraded) state */
state->baseline.lap_fast = ema_update(state->baseline.lap_fast, ...);

The fast EMA continues to update so that the ratio numerator tracks whatever the current scene looks like. The slow EMA stays pinned at the last known clean baseline. The ratio therefore stays depressed (or elevated) for as long as the degradation persists, no matter how many frames pass.

Startup Warmup (60 Frames)

At startup, both EMAs are at zero. It takes time for them to converge to the actual scene values. During this convergence window, the ratios are meaningless. We suppress all alarms for the first 60 frames (6 seconds at 10 Hz), letting both EMAs stabilise before any classification decisions are made.

After 60 frames, warmup_done is set to 1 and alarms are enabled. The baseline at this point represents a meaningful clean-scene reference. If the camera is mounted already degraded (e.g. lens already frosted), the baseline will converge to degraded values and the detector will not fire after warmup. This is a known trade-off — the solution is the optional NVM serialise/deserialise API, which allows a known-good baseline to be saved to flash and restored after a reboot, skipping the warmup entirely.

5Classification — The Decision Tree

With all seven ratio values computed, the classifier evaluates them in a fixed priority order. The order matters: more specific or stronger signals are checked first, preventing ambiguous conditions from being masked by the more general ones.

1
DIRT
r_occ > 192 (dark flat cells elevated >50% above baseline)
AND r_cellvar > 160 (frame is spatially non-uniform)

Dark opaque blobs on the lens create cells with near-zero variance and brightness well below the global mean. The combination of dark flat cells and high spatial non-uniformity is extremely specific to this condition.
2
OBSTRUCTION
r_obs > 192 (flat cells at any brightness, elevated >50% above baseline)
AND r_cellvar > 160 (spatially non-uniform)
AND r_occ ≤ 96 (occlusion score NOT elevated — rules out dirt)

Checked second because it uses a superset of the dirt condition (flat cells, any brightness). The r_occ ≤ 96 guard ensures we only reach this branch if the flat cells are NOT dark — i.e. tape, ice, or a bright obstruction rather than mud.
3
FROST
r_lap < 80 (sharpness dropped to ~62% of baseline)
AND r_mean > 160 (mean brightness elevated ~25% above baseline)
AND r_spr < 56 (histogram spread collapsed severely)
AND r_cellvar < 96 (uniform across all cells — no single-cell blob)

Requires all four conditions simultaneously. The elevated mean is the critical separator from fog — frost adds a bright diffuse veil; fog does not. The severely collapsed histogram (threshold lower than fog's) ensures we are seeing the extreme brightness compression characteristic of ice on the lens.
4
FOG
r_ctr < 80 (contrast dropped to ~62% of baseline)
AND r_lap < 80 (sharpness dropped)
AND r_mean > 96 AND r_mean < 160 (mean stays near baseline — not elevated)
AND r_cellvar < 96 (uniform across the frame)

The bounded mean check (96..160) is the key fog-frost discriminator. Fog keeps mean near its baseline; frost pushes it upward. The lower bound (> 96) also excludes dark scenes from being called fog — a completely dark frame has near-zero mean which drops the ratio below 96.
5
CLEAN
None of the above conditions triggered. The default safe output.
Note on the Python port

The Python implementation in detector.py uses absolute thresholds on raw feature values rather than ratios, because the Python version is primarily used for real-time testing on a PC before the C code is deployed to hardware. The C implementation uses the full ratio-based approach described above. The Python classifier is therefore more sensitive to ambient lighting conditions, which is why a bright-environment threshold like mean > 170 can fail in dim rooms — a known limitation that would be fixed by switching the Python classifier to use the same ratio approach.

6Hysteresis State Machine

The raw classifier output is noisy. A single frame of rain, a bird flying past, or a bright reflection can momentarily satisfy one of the degradation conditions without the lens actually being dirty. To prevent flickering alarms and false positives, a hysteresis state machine filters the raw output.

The state machine has two counters: confirm_count and clear_count.

Current stateRaw outputAction
CLEAN non-CLEAN Increment confirm_count. If confirm_count ≥ 5, switch to the detected degradation class and reset counter.
CLEAN CLEAN Reset confirm_count to 0. A single clean frame resets the whole confirmation window.
Degraded CLEAN Increment clear_count. If clear_count ≥ 15, switch back to CLEAN and reset counter.
Degraded Different degradation Switch type immediately. No hysteresis between degradation types — we want to report the most current classification without delay.

The asymmetry (5 frames to enter, 15 frames to clear) is intentional. It is much worse to miss an actual degradation for a few frames than to clear an alarm a second late. Lens contamination does not vanish instantly — if the detector sees 15 consecutive clean frames, the degradation has genuinely cleared.

At 10 Hz detection rate:

  • Enter degraded: 5 frames = 0.5 seconds
  • Clear to CLEAN: 15 frames = 1.5 seconds

7Memory Design

The entire module state lives in a single zero-initializable struct: DegradationState. Declaring it as a static variable at file scope initializes it to all-zeros at link time — no init function required.

The DegradationBaseline struct can be serialized to a byte buffer and saved to non-volatile memory (NVM/flash). On next power-up, deserializing it restores the clean-scene baseline and skips the 60-frame warmup, giving the detector immediate full-sensitivity operation from the first frame.

Call Sequence

/* 1. Application declares state once — zero-init is valid start */
static DegradationState cam_state;   /* .bss — zeroed at link time */

/* 2. Application calls this for each frame to analyse */
DegradationResult result;
degradation_update(&cam_state, frame_ptr, width, height, &result);

/* 3. Result contains the classification and confidence */

    raise_alarm(result.classification);
}

Inside degradation_update()

The function is structured in six sequential blocks — each a comment-delimited section in the source:

  1. Stack allocation — declares all working arrays (cell accumulators, histogram, etc.) as local variables. These live on the stack and are released when the function returns.
  2. Single-pass main loop — iterates over the subsampled frame, updating all accumulators simultaneously.
  3. Thumbnail MAD loop — a small separate loop over the 192-pixel thumbnail comparing current vs. previous frame.
  4. Feature derivation — computes the 8 final feature values from the accumulated raw data.
  5. EMA baseline update — updates fast and (conditionally) slow EMAs for each feature.
  6. Warmup check → Ratio computation → Decision tree → Hysteresis — the classification path, skipped during warmup.

Key Internal Helpers

/* EMA update — one cycle on Cortex-M0, no multiply/divide */
static uint16_t ema_update(uint16_t ema, uint8_t raw, uint8_t shift) {
    return ema - (ema >> shift) + raw;
}

/* Recover raw value from scaled EMA storage */
static uint8_t ema_raw(uint16_t ema, uint8_t shift) {
    return (uint8_t)(ema >> shift);
}

/* Ratio: fast vs slow EMA, 128 = neutral */
static uint8_t compute_ratio(uint8_t fast_raw, uint8_t slow_raw) {
    uint16_t num = (uint16_t)fast_raw * 128;
    uint16_t den = (uint16_t)slow_raw + 1;
    uint16_t r   = num / den;          /* single division per feature, outside hot path */
    return r > 255 ? 255 : (uint8_t)r;
}

10Known Limitations

1. Total Camera Occlusion (Hand, Lens Cap)

Covering the entire lens with a hand or placing a lens cap produces an all-black frame. All 12 grid cells become flat, but cell_mean_variance collapses to zero (every cell has the same near-zero mean) — failing the spatial non-uniformity gate. The global mean falls well below the fog floor threshold. Result: the frame is classified as CLEAN. The fix is a special total-occlusion branch: if the vast majority of cells are flat AND the global mean is extremely low, classify as OBSTRUCTION regardless of spatial non-uniformity.

2. Absolute Thresholds in the Python Port

The Python classifier uses fixed absolute thresholds (e.g. mean > 170 for frost) rather than the ratio-based approach in the C code. This means it is sensitive to ambient light levels and camera characteristics. In dim environments the brightness threshold is already satisfied in clean conditions, or never satisfied even in frost conditions. The correct fix is to pass the slow-EMA baseline mean into the Python classifier and switch to a relative check: is the current mean elevated above the clean baseline?

3. Frost Proxy Materials in Testing

Bubble wrap does not optically simulate frost — it magnifies and distorts the scene through discrete bubble-lenses while keeping edge sharpness high. Good frost proxies are petroleum jelly on flat glass, a fogged pane (breathe on it), or a sheet of tracing paper, all of which create genuine diffuse uniform scattering without structural edges.

4. MJPEG Compression Artifacts

Many USB webcams compress frames internally with MJPEG before transmitting. The 8×8 DCT block boundaries appear as weak edges to the Laplacian filter, artificially inflating laplacian_var and making it harder to trigger the sharpness-drop condition for frost and fog. Area-downsampling the frame before processing would average out these block artifacts, at the cost of slightly coarser edge detection.

5. Static Clean Scenes and Temporal MAD

A camera mounted on a tripod pointed at a static wall produces the same low temporal MAD as a frosted camera. The dual-EMA baseline mitigates this — a static scene has a consistently low MAD baseline, so the ratio stays near 128 and no alarm fires. However, if a camera transitions from a moving scene to a genuinely static clean scene at the same time as frost forms, the two effects can partially cancel in the MAD signal. The other features (sharpness, contrast, histogram) provide the primary signal in this case.
