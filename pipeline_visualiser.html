A note on "wrong" classifications: this is an absolute-threshold degradation detector calibrated for one automotive road camera — not a general image classifier. Bright/smooth photos read FROST, low-contrast photos read FOG, and sky-over-ground scenes read OBSTRUCTION because that is what the real thresholds do on those pixels. The sensor-noise σ control models the read-noise floor a real sensor always has (a downsampled JPEG has none), which removes the most common false FROST/FOG on ordinary photos.

1Input → 8-bit grayscale frame

The detector consumes a row-major uint8 grayscale buffer — no colour ever reaches the algorithm. The uploaded image is cover-cropped to 4:3, resampled to 320×240, and converted with integer BT.601 luma: gray = (R·4899 + G·9617 + B·1868 + 8192) >> 14.

2Single pass over the frame · stride 4

One loop visits every 4th pixel in x and y (1/16th of all pixels ≈ 4 800 samples), starting at (1,1) and stopping one pixel short of each edge so the 3×3 Laplacian never needs a bounds check. Every sample simultaneously feeds four accumulators: global min/max/sum, the 8-bin histogram, the Laplacian sharpness sum, and its 4×3 grid cell.

4Feature derivation

The accumulators collapse into seven uint8 features (plus the two temporal ones). The histogram_spread card replays the 5%→95% percentile walk on its own mini histogram.

↳ feature derivation — degradation.c:218-354

5Absolute-threshold decision tree · priority order

Raw feature values are tested against fixed thresholds from degradation.h — no baseline, no ratios. Rules run top-down; the first match wins and the rest are skipped. The first three rules are the shipped degradation.c tree; the two OBSTRUCTION rescue rules below them are the live-viewer override from live.py, reached only when the core tree would return CLEAN.
